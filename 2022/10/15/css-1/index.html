CSS基础

CSS是层叠样式表(Cascading Style Sheets)的简称，也称CSS样式或级联样式表，是一种标记性语言。
CSS主要用于网页版面的布局，控制外观显示样式等。

引入方式

  1. 内部样式表(嵌入式)
    HTML与CSS放在同一个HTML页面中，一般放在head标签中，方便控制当前页面样式设置。
    <style type="text/css">CSS样式</style>
  2. 行内样式表(行内式)
    在元素标签内部的style属性中设置CSS样式，适合简单修改样式。
    <div style="color: red; font-size: 20px;">文字内容</div>
  3. 外部样式表(链接式)
    将CSS样式单独写在CSS文件中，再通过链接引入HTML文档中使用。
    结构和样式完全分离，可维护性强，最理想的方式。
    <link rel="stylesheet" type="text/css" href="css/index.css" />

选择器

基础选择器

基础选择器是由单个选择器构成的，可分为标签选择器、类选择器、id选择器和通配符选择器。

  1. 标签选择器选择器 = 标签名
  2. 类选择器选择器 = .类名结合元素class属性使用，同一页面中相同类的元素可以有多个。
  3. id选择器选择器 = #id结合元素id属性使用，每个id在每个HTML文档中只出现一次。
  4. 通配符选择器选择器 = *表示选取页面中所有元素，特殊情况下使用(如清除页面所有元素某个样式)。

复合选择器

复合选择器由两个或多个基础选择器组合而成。

  1. 并集选择器 元素1,元素2 {样式声明}
    表示同时选择元素1和元素2，为他们定义相同的样式。
  2. 伪类选择器
    选择器 说明
    a:link 选择所有未被访问的链接
    a:visited 选择所有已被访问的链接
    a:hover 选择鼠标指针位于其上的链接
    a:active 选择活动链接(鼠标按下未弹起的链接)
    书写顺序应按照LinkVisitedHoverActive书写，否则可能会失效。
    input:focus 选择获得焦点的表单输入框
  3. 层次选择器
    选择器 说明
    M N 后代选择器，选择M元素内所有后代N元素
    M>N 子代选择器，选择M元素内所有子级N元素
    M~N 兄弟选择器，选择M后同级的所有N元素
    M+N 相邻选择器，选择同级的M后的下一个相邻的N元素
  4. 结构选择器
    第一组
    结构选择器 示例 说明
    :first-of-type p:first-of-type 父级下第一个p标签
    :last-of-type p:last-of-type 父级下最后一个p标签
    :nth-of-type(n) p:nth-of-type(2) 父级下第二个p标签
    :nth-last-of-type(n) p:nth-last-of-type(3) 父级下倒数第三个p标签
    第二组
    结构选择器 示例 说明
    :first-child span:first-child 父级下第一个span子元素
    :last-child span:last-child 父级下最后一个span子元素
    :nth-child(n) span:nth-child(2) 父级下第二个span子元素
    :nth-last-child(n) span:nth-last-child(2n-1) 父级下倒数的奇数span子元素
    说明：
    • 关于n：n从1开始，n可以取值为2n、3n+1、even、odd等。
    • 关于类名：
      :nth-child(n)会先选中第n个子元素，且该子元素拥有此类时才会应用样式；
      :nth-of-type(n)会先选中每一类型标签下的第n个元素，且当被选中的元素拥有此类时才会应用样式。

三大特性

层叠性

相同选择器设置相同的样式，此时一个样式会覆盖另一个冲突的样式。
哪个样式离结构近，就执行哪个样式。

继承性

子标签会继承父标签的某些样式，如颜色、字号和行高等。恰当使用可降低代码复杂性。

优先级

选择器 选择器权重(a,b,c,d)
继承或者* 0,0,0,0
元素选择器 0,0,0,1
类/伪类选择器 0,0,1,0
id选择器 0,1,0,0
内联样式 1,0,0,0
!important 无穷大

权重比较：从左往右依次，较大的权重大，如果相同，则继续往右移动比较。
权重叠加：复合选择器使用时间会有权重叠加，需要计算权重，但是权重不会进位。

字体属性

属性 说明 备注
font-size 字号 取值数字+px或者关键字(small/medium/large)
font-family 字体 多字体从前往后依次采用，”字体1”,”字体2”,…,”字体n”
font-weight 字体粗细 取值100~900或关键字(lighter/normal/bold)
font-style 字体样式 斜体效果，取值italic/oblique
font 字体连写 连写顺序不能更换，必须写字号和字体

font连写顺序：font: font-style font-weight font-size/line-height font-family

文本属性

属性 说明 备注
color 文本颜色 取值为关键字、十六位进制或RGB值
text-align 文本对齐 取值为left/center/right (对inline/inline-block元素均有用)
text-indent 文本缩进 取值为像素值，两字距缩进：2em或字号的2倍px
text-decoration 文本修饰 取值none(无划线)/underline(下划线)/line-through(中划线)/overline(顶划线)
text-transform 文本大小写 取值none(无)/uppercase(大写)/lowercase(小写)/capitalize(首字母大写)
text-spacing 字间距 取值为像素值(每个英文字母视为一个”字”)
word-spacing 词间距 取值为像素值(每个英文单词)
line-height 行间距 取值为像素值
vertical-align 垂直对齐 取值为top/middle/bottom/baseline(基线)

背景属性

属性 说明 值
background-color 背景颜色 取值为关键字、十六位进制或RGB值
background-image 背景图片 取值为url(图片路径)
background-repeat 是否平铺 repeat/no-repeat/repeat-x/repeat-y

CSS盒子模型

在CSS中，所有HTML元素可以看作盒子，一个用盒模型（box model）封装的HTML元素，包括边距，边框，填充，和实际内容四个部分。

盒子模型

在标准盒模型(content-box)中，为元素设置width和height实际是设置content的宽度和高度。

边框(border)

属性 说明 值
border-width 边框大小 取值为像素值
border-style 边框样式 solid(实线)/dashed(虚线)/dotted(点线)
border-color 边框颜色 取值为颜色取值
border 边框简写形式 width style color
border-collapse 边框合并 separate(默认值，独立边框)/collapse

取消边框：border: 0或者border: none，两者的区别在于有渲染或没渲染。

内边距(padding)

  1. 内边距可以每个方向单独指定：padding-left、padding-right、padding-top、padding-bottle
  2. 内边距也可以简写，根据值的多少应用：
    padding: 1个值(四周)/2个值(上下&左右)/3个值(上&左右&下)/4个值(上右下左)
  3. 一般情况下，如果盒子本身没有指定width/height，padding不会撑大盒子。

外边距(margin)

  1. 外边距的设置方式与内边距完全相同，包括单独设置和简写方式。
  2. margin相关：
    相关情况 说明
    margin塌陷 父子关系的块元素，同时有外边距时，父元素会取两者中较大的
    margin合并 兄弟关系的块元素，同时有外边距时，相邻处外边距会取两值中较大的
    负margin 设置负值的margin时，元素会被“拉”向负值方向

块级盒子与内联盒子

特性对比

块级盒子(block) 内联盒子(inline)
块级盒子独占一行 相邻内联盒子占一行，一行内可以有多个
高度、宽度、内外边距可以控制 高度，宽度直接设置无效，默认宽度是本身内容宽度
内部可以放其他块级盒子和内联盒子 内部只能容纳其他内联盒子
div、p、dt、h1等标签默认都是block盒子
但p、dt和h1等标签内不能容纳其他块级盒子		 a标签、span标签默认是inline盒子

行内块元素

行内块元素(inline-block)，又称可变元素，兼顾block盒子和inline盒子特点：

  • 默认宽度为内容宽度，一行内可以放置多个；
  • 可以指定宽度、高度和内外边距；
  • img、input、textarea默认显示为inline-block。

display属性

通过display属性可以指定元素的显示方式。

display: none 隐藏元素后不占用原来的位置；
visibility: hidden 隐藏元素后占用原来的位置。

CSS浮动

正常文档流

正常布局流、标准流(normal flow)：

  • 默认的布局方式。
  • 块级元素独占一行，按书写顺序从上往下放置。
  • 内联元素与其他内联元素和文本同处一行，从左往右依次排满，如果一行空间不够则换行。

浮动与特性

浮动(float)：创建浮动框，将其移动到一边直至碰到块元素或另一个浮动框的边缘。
特性：

  • 浮动的元素会脱离标准流(脱标)，移动到指定位置，并且不再保留原先的位置。
  • 浮动的元素会在一行内显示并且元素顶部对齐。
  • 浮动的元素会具有行内块元素的特性。
  • 浮动的元素只影响后面的标准流，不会影响前面的标准流。

布局说明

  • 多个块级元素，纵向排列标准流，横向排列用浮动。
  • 一个盒子里有多个盒子，如果其中一个盒子浮动了，其他兄弟也应该浮动。
  • 为了防止浮动带来的影响，使用浮动后应清除浮动。

清除浮动

浮动元素不再占用原文档流的位置，会对后面的元素排版产生影响，为了解决这些问题，就需要清除浮动。

CSS定位

CSS元素定位由定位模式与边偏移两个部分组成。
定位模式：static、relative、absoulute、fixed、sticky
边偏移：top/bottom/left/right，指定位元素相对父元素边线的距离。

定位模式

  1. 静态定位(static)
    元素默认定位方式，没有边偏移。
  2. 相对定位(relative)
    • 元素在移动位置时，是以原来的位置为参考点。
    • 不脱标，仍然以标准流的方式占用原来位置。
  3. 绝对定位(absolute)
    • 元素无祖先元素或祖先元素无定位，以浏览器为参考点。
    • 元素祖先元素有定位，则以最近一级有定位的祖先元素为参考点。
    • 绝对定位元素脱标。
  4. 固定定位(fixed)
    • 元素固定于浏览器可视区位置，浏览器页面滚动时，元素位置不变。
    • 固定定位元素脱标，不占原先位置，跟父元素定位无关。
  5. 粘性定位(sticky)
    • 元素以浏览器可视窗口为参照点移动。(固定定位特点)
    • 粘性定位元素占用原先的位置。(相对定位特点)
    • 必须添加left/right/top/bottom其中一个才有效果。

定位的特点

  • 脱标的定位元素有行内块元素的性质，脱标的定位元素不会触发外边距塌陷。
  • 定位的盒子有z-index属性，越大的层级可以压住小的层级。
  • 浮动的盒子不会压住下面盒子中的内容，定位的盒子会压住下面盒子的内容。

BFC及应用

BFC概念

块格式化上下文（Block Formatting Context，BFC）是 Web 页面的可视 CSS 渲染的一部分，是块级盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。
简单来说，具有BFC特性的元素可以看作是隔离了的独立容器，容器里面的元素不会在布局上影响到外面的元素，并且BFC具有普通容器所没有的一些特性。

BFC特点

  1. BFC区域相当于一个容器，内部元素不会影响到外部，外部元素也不会影响到内部。
  2. 同一个BFC区域内，外边距会发生折叠。
  3. BFC区域内的元素不会与浮动元素重叠。
  4. 计算BFC区域的高度时，浮动元素也参与计算。

创建BFC

  1. 根元素(html)(html所在区域即为BFC区域，因此默认整个网页都在BFC内渲染)
  2. 浮动元素(float不是none)
  3. 绝对定位元素(position为absolute或fixed)
  4. 行内块元素(inline-block)、弹性元素(flex、inline-flex)、网格元素(grid、inline-grid)等…
  5. overflow不为visible的块元素
  6. 其他

BFC应用

  1. 消除margin塌陷

    父子元素的margin塌陷也可以通过给父元素设置overflow属性来解决。

  2. 清除浮动

    父元素高度被浮动盒子撑开

  3. 防止浮动元素重叠

布局整体思路

先结构，后样式

  1. 测量确定确定页面版心(可视区)。
  2. 分析页面行模块以及每个行模块中列模块。(布局第一准则)
  3. 分析确定每个列模块大小，再确定列模块位置。(布局第二准则)
  4. 制作HTML结构，确定各盒子在页面的位置。(普通流+浮动+定位)
  5. 确定结构后，设置盒子样式，往盒子中添加内容。
