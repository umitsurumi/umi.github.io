# 数组

# 数组简介

Java 中的数组用于存储多个相同数据类型的元素，是一种引用数据类型，存储的数组元素可以是基本数据类型，也可以是引用数据类型。
定义一个数组变量的语法为 数据类型[] 标识符 或 数据类型 标识符[] 。定义完成数组变量之后需要对数组进行初始化，所谓初始化，即为数组在内存中开辟一块连续的内存空间用于存储数据，数组元素在内存中是紧密有序的排列的。与 JavaStript 不同，Java 数组一旦创建，其大小就不可改变。

在定义数组的同时为其赋予初始值叫静态初始化，而先声明数组长度，之后再为其赋值的是动态初始化。采用动态初始化声明的数组，Java 会根据声明的数组类型自动为其分配初始值，如：布尔数组元素初始值为 false ，整型数组元素初始值为 0 ，浮点型数组元素初始值为 0.0 ，引用类型数组元素初始值为 null 。

# 数组的存储

Java 的数组为引用数据类型，数组标识符中存储的是首个数组元素的内存地址（可以看成是一个指针），这个地址值存储在栈内存中。而数组元素占有一片连续的内存空间，存储在堆内存中。

二维数组也是类似，只不过在二维数组中，每一个数组元素又是指向另一个数组的地址值。

因此，当使用赋值符号 = ，实际上是复制了数组的地址，而非真的复制了数组元素。当希望复制一个数组，需要使用遍历的方法，对数组的每一个元素都进行复制。

也是因为同样的原因，两个数组直接使用比较运算符 == ，进行比较的是它们的地址值，而非它们的元素，即使两个数组元素相同，也会返回 false 。

因此，为了便于进行常见的数组操作，Java 提供了一个 Arrays 工具类，它包括了数组排序、数组查找（二分查找）、数组比较和数组复制（浅拷贝）等操作。

# 字符串

# String 简介

字符串在 Java 中是一种引用数据类型，Java 提供了 String 类来创建和操作字符串。

两种方法的区别在于，使用 "" 方法给出的字符串存储在字符串常量池中，并且相同的内容只会在其中存储一份。但是通过构造方法 new 创建，每次都会产生一个新对象，存储在堆内存中。

Java 通过这种方法，可以：

  1. 保证 String 对象的安全性，避免 String 被更改。
  2. 保证 hash 值不会频繁变更。
  3. 可以实现字符串常量池。

当使用 "" 创建字符串时，JVM 会先检查对象是否存在字符串常量池中，如果存在则返回该对象的引用。这样可以减少同一个值的字符串对象重复创建，节约内存。
当使用 String str = new String("Hello") 方法时， "Hello" 常量字符串会先放到常量结构中，类加载的时候， "Hello" 会在常量池中创建。当调用 new 时，JVM 会调用 String 的构造函数，同时引用常量池中的 "Hello" 字符，在堆内存中创建一个 String 对象，然后 str 引用 String 对象。

JDK6 中，字符串常量池在方法区，JDK7 开始，字符串常量池移动到了堆空间。

# String 性能考量

在字符串拼接时，字符串常量进行的拼接，结果仍然存储在字符串常量池中，当有变量参与的时候，结果存储在堆中。

JDK8 中，在字符串变量拼接时，编译器实际上优化成了 StringBuilder 的方式：

JDK8 及以后，Java 会根据变量的长度预估计算后的长度，然后创建一个数组，最后将这个数组变为一个字符串。

不论底层是哪种方法进行的字符串拼接，最终都会 new 一个 String 对象，因此最终的地址值不同，使用 == 判断结果是 false 。
正确的字符串的比较，应该使用 equals() 方法。

虽然可以使用 + 直接拼接字符串，但是这种方法会在内存中生成大量临时的字符串对象，不但浪费内存，也影响了程序性能和 GC 效率。正确的拼接方案应该优先考虑用 StringBuilder 的 append() 方法来替代 + 。

StringBuilder 是一个可变对象，可以预分配缓冲区，在往其中新增字符时，不会创建新的临时对象。另外， append() 方法会返回 this ，可以实现链式调用。
在并发编程中， String 对象的拼接涉及到线程安全时，可以考虑使用 StringBuffer 对象，它通过同步来保证多线程操作的安全。因为同步会带来执行速度的下降，与 StringBuilder 相比，它的性能稍差。
