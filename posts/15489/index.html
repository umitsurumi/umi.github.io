# 响应式布局

随着移动互联网的发展，为了适应各种智能设备不同屏幕尺寸和窗口大小，响应式布局应运而生。

# 布局比较

# 静态布局 (Static Layout)

传统 web 布局方式，网页上元素以 px 作为单位。
特点：不管浏览器尺寸多少，网页始终按照最初书写的布局来显示，超出部分会在浏览器上显示滚动条。

# 流式布局 (Liquid Layout)

网页中主要区域的划分使用百分数，页面元素按屏幕分辨率调节尺寸。
特点：页面整体布局结构不变，元素大小会随屏幕大小缩放，元素缩放后可能会出现不协调的情况。

# 自适应布局 (Adaptive Layout)

结合媒体查询，为不同尺寸和介质的设备设置不同的样式。
特点：创建多个静态布局，分别为不同屏幕分辨率定义布局，屏幕分辨率变换时，元素位置变化而大小不变。

# 响应式布局 (Responsive Layout)

目的是为了确保一个页面设计在所有终端 (pc + 移动端或其他大小的屏幕) 都有较好的显示效果。
通常的实现方法是流式布局 + 弹性容器结合媒体查询。
特点：在不同的屏幕分辨率下有不同的布局样式，元素大小和位置都会改变。

# 设计基础

# viewport

viewport 是用户在网页上的可见区域。HTML5 引入了一种方法，可以通过 meta 标签来控制可视区域：

  • width=device-width 将页面宽度设置为跟随设备屏幕宽度。
  • initial-scale=1.0 设置首次加载页面时的初始缩放级别。

因为通常情况下，移动端浏览器默认为 980px 的设计宽度。
通过引入这段 meta 标签，取消浏览器的缩放，来启用响应式设计。
