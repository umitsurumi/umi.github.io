# 泛型

泛型（generics）是 JDK5 引入的新的特性，它允许在定义类、接口和方法时，通过一个标识表示其中某个参数的类型，这个类型参数将在使用时确定。
泛型提供了一种编译时的类型安全检测机制。在没有泛型之前，Java 提供了对 Object 类型应用的任意化操作，即对 Object 的转型操作。但在类型转换的过程中，向上转型是安全的，向下转型却可能导致 ClassCastException 异常，泛型的出现可以预防这种问题：

另外，对于静态方法来说，因为 Java 中的泛型类型参数是在实例化时确定的，对于静态方法来说，它是属于类而不是属于对象的，如果要在静态方法中使用泛型类型，则需要单独指定。

# 类型擦除

Java 中的泛型是使用类型擦除来实现的，使用泛型时，任何具体的类型信息都被擦除了。
JVM 对泛型一无所知，例如 Demo<String> 和 Demo<Integer> ，运行时都将它们视为相同的类型。
编译器将泛型的 <T> 都视为了 Object ，同时再根据 <T> 来实现安全的强制类型转换：

因此，Java 的泛型是由编译器在编译时实行的，泛型虽然被视为 Object ，但是在需要转型的时候编译器会自动实现安全的强制类型转换。类型擦除中没有产生新的类型，所以泛型不会产生运行时的开销。

由于这种类型擦除的机制，Java 的泛型存在这些局限：

  1. 泛型不能是基本数据类型，因为 Object 无法持有基本数据类型；
  2. 因为类型擦除，所以无法取得泛型的 Class ，也无法判断泛型的类型；
  3. 不能实例化泛型 T ，也不能进行转型、 instanceof 等操作。

符号 ? 称为类型通配符，用来代替具体的类型参数。如 Demo<?> 在逻辑上可以是 Demo<String> 、 Demo<Number> 等 Demo<具体类型> 的父类。
类型通配符可以结合 extends 和 super 关键字用于限定类型的上届和下界，两者不能同时使用。

  • <? extends A> 上界通配符：未知类型必须为 A 或它的子类类型。
  • <? super B> 下界通配符：未知类型必须为 B 或它的父类类型。

在一些写法上，可以认为 <T extends OtherClass> 和 <? extends OtherClass> 有类似的地方，但是两者最大的区别在于， T 可以代表一个实际的类型，而 ? 只是一个通配符。因此 T 可以用在泛型类型、泛型方法的声明上，而 ? 在这些地方是不能使用的。

# 集合

集合可以看作是一个用来存储对象的容器。
与数组不同，集合不能存储基本数据类型（只能存储它们的包装类型），但是它的大小可变，配合 Java 提供的操作集合的相应方法，能更方便地对存储的对象进行操作。。

# 概述

Java 中的集合主要有两种， Collection 和 Map 。
Collection ：用于存储单个的数据，也称为单列数据集合。
Map ：用于储存具有映射关系的 “键值对”（key-value），也称为双列数据集合。

迭代器 Iterable 和 Iterator 的目的在于便利访问集合中的元素， Collection 接口继承自 Iterable 接口。

Collection 接口继承并扩展了 Iterable 接口，该接口有一个 iterator 方法，用以返回一个实现了 Iterator 接口的对象。
iterator 返回的迭代器对象，默认游标在集合的第一个元素之前（-1），其中， hasNext() 方法用于判断是否还有元素可以迭代， next() 方法用于返回迭代的下一个元素，如果没有可达到的元素会抛出 NoSuchElementException 异常。
可以使用增强的 for 循环来遍历 Collection ，语法为 for(E e : Collection<E>) ，实际上在循环时就是使用的迭代器进行的。
另外，在进行迭代的时候，不要随意更改集合长度，会导致并发修改异常。

# Collection

Collection 接口是单列数据集合的顶级接口，Java 并没有对它进行直接实现，而是实现了更具体的子接口。
Collection 接口中的一些常用方法如下：

方法说明
int size()返回当前集合中存储元素个数
boolean isEmpty()判空操作，返回布尔值
boolean add(E e)添加元素 e 到当前集合中
boolean addAll(Collection<? extends E> c)添加集合 c 中的所有元素对象到当前集合中
boolean remove(Object o)从当前集合中删除第一个与 o 进行 equals() 判断相等的元素
boolean removeAll(Collection<?> c)从当前集合中删除所有与集合 c 中 equals() 判断相等的元素
boolean removeIf(Predicate<? super E> filter)从当前集合中删除 filter 判断为 true 的元素
boolean contains(Object o)当前集合中是否存在一个与 o 进行 equals() 判断相等的元素
boolean containsAll(Collection<?> c)集合 c 中的所有元素是否都在当前集合中
boolean retainAll(Collection<?> c)保留同时存在当前集合和 c 集合中的元素（取交集）
void clear()清空当前集合元素
Object[] toArray()返回包含当前集合中所有元素的数组

Java 中还提供了一个 Collections 工具类，这个工具类提供了一些列的静态方法，用于操作集合：

  • 排序：

    方法说明
    reverse(List)反转列表元素的顺序
    shuffle(List)对列表元素进行随机排序
    sort(List)对列表元素按自然顺序进行升序排序
    sort(List, Comparator)对列表元素按指定比较器排序
    swap(List, i, j)交换列表 i 和 j 处的元素

  • 查找：

    方法说明

    Object max(Collection, Comparator)根据 Comparator 指定顺序，返回最大值

    方法说明

  • 同步：
    Collections 类提供了多个 synchronizedXxx() 方法，可以将对应的集合包装成线程同步的集合，可以解决多线程并发时的安全问题。

# List

List 接口是 Collection 的子接口，其中扩展了一些根据索引操作集合元素的方法。

方法说明
void add(index,element)在索引 index 的位置插入元素
boolean add(index, elements)在索引 index 将元素全部插入
Object get(index)获取索引 index 的元素
List subList(from, to)返回从 from 到 to 索引位置元素组成的子列
int indexOf(element)获取元素首次在列表中出现的索引
int lastIndexOf(element)获取元素最后一次在列表中出现的索引
Object remove(index)删除指定索引位置的元素
Object set(index, element)设置指定索引位置的元素

List 有三个实现类，分别是 ArrayList 、 LinkedList 和 Vector 。

# ArrayList

ArrayList 从底层数据结构角度来看是一个动态数组。

  1. 构造方法：

    • public ArrayList(int initialCapacity)
    • public ArrayList()
    • public ArrayList(Collection<? extends E> c)

    ArrayList 可以从集合构造，也可以指定初始容量。当没有指定初始容量时， ArrayList 的默认容量大小为 10，这个操作发生在没有指定初始容量的 ArrayList 对象第一次调用 add() 方法时。
    在使用 add() 方法添加元素时，如果当前容量已满，会自动扩容为当前容量的 1.5 倍。因此，在初始化时为其指定合适的容量大小，可以减少扩容的性能开销。

  2. 特点

    • 元素有序，元素可重复
    • 底层是数组，有索引值，查找快，增删慢（涉及到数组元素的移动）
    • 实现了 Cloneable 接口，默认为浅拷贝
    • 实现了 Serializable 接口，支持序列化
    • 线程不安全

# LinkedList

LinkedList 从底层数据结构角度来看，可以视为双链表。

  1. 特有方法

    • public void addFirst / addLast(E e) ：将指定元素插入列表的开头 / 结尾。
    • public E getFirst / getLast() ：将获得列表开头 / 结尾的元素。
    • public E removeFirst / removeLast(E e) ：移除列表开头 / 结尾的元素。
    • public E pop() ：从列表表头出栈一个元素。
    • public void push(E e) ：从列表表头入栈一个元素。

    LinkedList 在内部维护了一个双链表，通过 Node 类型的头尾指针（ first 和 last ）来访问数据。 Node 是它的内部类， prev 是该结点的上一个结点， next 是该结点的下一个结点， item 是该结点包含的值。
    LinkedList 对查找做了一些优化。 index 索引，如果是在链表前半部分，则会从头结点开始遍历，如果是在链表后半部分，则从尾结点开始遍历，但是最坏情况下仍然会扫描N/2N/2N/2 个元素。

  2. 特点

    • 元素有序，元素可重复
    • 底层是双链表，有索引值，查找慢（链表只能遍历），增删快（增删操作不需要移动元素）
    • 支持序列化，并且针对自身数据特征进行了优化
    • 线程不安全

# Vector

Vector 和 Stack 的设计目标是作为线程安全的 List 实现，替代 ArrayList 。

  • Vector ： Vector 和 ArrayList 类似，也实现了 List 接口。但是， Vector 中的主要方法都是 synchronized 方法，即通过互斥同步方式保证操作的线程安全。 Vector 无参构造时初始容量为 10，自动扩容时扩容 2 倍。
  • Stack ： Stack 也是一个同步容器，它继承于 Vector 类，它的方法也用 synchronized 进行了同步。

# Map

Map 是双列数据集合的顶级接口。和 Collection 接口不同， Map 中的数据是以键值对（ key-value ）形式存储的。
Map 中不允许有重复的 key ，每个 key 只允许映射到一个 value ，但是 value 的值可以重复。
在 JDK8 之前， Map 的底层实现采用数组 + 链表的形式；JDK8 之后，优化为了数组 + 链表 + 红黑树。
Map 接口的一些常用方法如下：

方法说明
V put(K key, V value)将指定的 key-value 添加到当前 map 对象中，如果 key 值重复，则覆盖之前 value
void putAll(Map<K, V> m)将 m 中的所有 key-value 对存放到当前的 map
V remove(Object key)移除指定 key 的 key-value 对，返回 value
void clear()清空当前 map 中的所有数据
V get(Object key)获取指定 key 对应的 value
boolean containsKey(Object key)是否包含指定的 key 值
boolean containsValue(Object value)是否包含指定的 value 值
int size()返回 map 中 key-value 对的个数
boolean isEmpty()判断当前 map 是否为空
boolean equals(Object obj)判断当前 map 和对象 obj 是否相等
Set<K> keySet()返回所有 key 构成的 Set 集合
Collection<V> values()返回所有 value 构成的 Collection 集合
Set<Map.Entry<K, V>> entrySet()返回所有 key-value 构成的 Set 集合

# HashMap

HashMap 是 Map 接口最常用的实现类。

  1. 特点
    • 无序，无索引， key 值唯一
    • 可以存储 null 键值
    • 线程不安全
  2. 说明
    • 判等条件：对 key ， HashMap 先判断它们的哈希值，哈希值不相等的情况下再判断它们 equals() 是否相等。对 value ，直接判断它们 equals() 是否相等。
    • 数据结构：
      • HashMap 使用 Node<K,V>[] 数组来存储元素，当第一次存储数据时，如果没有指定初始容量，会创建一个默认容量为 16 的数组。
      • 构造时可以指定初始容量和加载因子，但是初始容量并不直接等于传递的参数，而是根据传递的参数计算得出。
      • 如果没有指定加载因子，那么它的默认值为 0.75，当数组存储达到了加载因子限度时扩容 2 倍。
      • 如果元素哈希值相同，但是内容不同，会在同一个数组索引位上以链表的形式存储，当链表过长时，会转为红黑树优化查询效率。
      • 当链表长度大于 8，数组容量大于 64 时，链表转为红黑树；如果后续删除元素使链表长度小于 6，红黑树变回链表。

# LinkedHashMap

LinkedHashMap 是 HashMap 的子类，与 HashMap 的最大区别在于它按照元素插入顺序存储。
LinkedHashMap 维护了一个保存所有条目的双向链表，保证了元素迭代的顺序。

# TreeMap

TreeMap 基于红黑树进行实现，其内部元素是根据 key 来进行排序的。
在创建 TreeMap 对象时，可以传入一个 Comparator 对象， key 值会根据这个比较器来进行排序。如果没有这个比较器对象，则要求 key 值的类型实现了 Comparable 接口， TreeMap 对象会按照这个自然排序规则进行排序。
当两个 key 值按 compareTo() 方法或经 compare() 方法返回 0 时， TreeMap 认为两个 key 值相等。

# Hashtable

Hashtable 是 Map 的线程安全实现类，它底层也采用哈希表（数组 + 链表）的形式实现。与 HashMap 不同的是，它不允许 null 的键值，并且效率较 HashMap 低，现在已经不常使用。

# Properties

Properties 继承自 Hashtable ，与 Hashtable 不同， Properties 现在仍然被用于处理属性文件。
Properties 的 key-value 要求都是字符串类型。
特有方法：

  • setProperty(String key,String value) ：存入键值对
  • getProperty(String key) ：根据 key 获取 value
  • Set<String> stringPropertyNames ：获取所有的 key 放到 Set 集合中。
  • void load(InputStream inStream) ：将流中的数据加载到 Properties 中。

# Set

Set 接口继承了 Collection 接口，两者提供的方法完全相同，只是在行为上有区别。
Set 中的元素是唯一的，它不允许有重复的元素。
Set 集合功能的实现，底层都是依靠 Map 集合实现的。

# HashSet

HashSet 是最常用的 Set 的实现类

HashSet 特点：

  • 元素无序，无索引，元素不可重复，允许 null 值
  • 实现了 Cloneable 接口，默认为浅拷贝
  • 实现了 Serializable 接口，支持序列化
  • 底层是数组，有索引值，查找快，增删慢（涉及到数组元素的移动）
  • 线程不安全

HashSet 是基于 HashMap 实现的， HashSet 的许多方法都依靠 HashMap 。它的底层数据结构在 JDK8 之前是数组 + 链表，JDK8 之后优化为数组 + 链表 + 红黑树。

# LinkedHashSet

LinkedHashSet 是 HashSet 的子类，与 HashSet 的最大区别在于它按照元素插入顺序存储。
LinkedHashSet 维护了一个保存所有条目的双向链表，保证了元素迭代的顺序。
LinkedHashSet 的构造方法实际上是调用的是父类 HashSet 的非 public 构造方法。

# TreeSet

TreeSet 的实现依赖于 TreeMap ，它底层的数据结构也是红黑树。
TreeSet 中的元素是有序的，它依靠自然排序顺序或指定的比较器顺序排序。
