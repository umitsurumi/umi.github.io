# CSS 基础

CSS 是层叠样式表 (Cascading Style Sheets) 的简称，也称 CSS 样式或级联样式表，是一种标记性语言。
CSS 主要用于网页版面的布局，控制外观显示样式等。

# 引入方式

  1. 内部样式表 (嵌入式)
    HTML 与 CSS 放在同一个 HTML 页面中，一般放在 head 标签中，方便控制当前页面样式设置。
    <style type="text/css">CSS样式</style>
  2. 行内样式表 (行内式)
    在元素标签内部的 style 属性中设置 CSS 样式，适合简单修改样式。
    <div style="color: red; font-size: 20px;">文字内容</div>
  3. 外部样式表 (链接式)
    将 CSS 样式单独写在 CSS 文件中，再通过链接引入 HTML 文档中使用。
    结构和样式完全分离，可维护性强，最理想的方式。
    <link rel="stylesheet" type="text/css" href="css/index.css" />

# 选择器

# 基础选择器

基础选择器是由单个选择器构成的，可分为标签选择器、类选择器、id 选择器和通配符选择器。

  1. 标签选择器 选择器 = 标签名
  2. 类选择器 选择器 = .类名 结合元素 class 属性使用，同一页面中相同类的元素可以有多个。
  3. id 选择器 选择器 = #id 结合元素 id 属性使用，每个 id 在每个 HTML 文档中只出现一次。
  4. 通配符选择器 选择器 = * 表示选取页面中所有元素，特殊情况下使用 (如清除页面所有元素某个样式)。

# 复合选择器

复合选择器由两个或多个基础选择器组合而成。

# 并集选择器

元素1,元素2 {样式声明}
表示同时选择元素 1 和元素 2，为他们定义相同的样式。

# 伪类选择器

选择器说明
a:link选择所有未被访问的链接
a:visited选择所有已被访问的链接
a:hover选择鼠标指针位于其上的链接
a:active选择活动链接 (鼠标按下未弹起的链接)

书写顺序应按照 LinkVisitedHoverActive 书写，否则可能会失效。
input:focus 选择获得焦点的表单输入框

# 层次选择器

选择器说明
M N后代选择器，选择 M 元素内所有后代 N 元素
M>N子代选择器，选择 M 元素内所有子级 N 元素
M~N兄弟选择器，选择 M 后同级的所有 N 元素
M+N相邻选择器，选择同级的 M 后的下一个相邻的 N 元素

# 结构选择器

  1. 第一组

    结构选择器示例说明
    :first-of-typep:first-of-type父级下第一个 p 标签
    :last-of-typep:last-of-type父级下最后一个 p 标签
    :nth-of-type(n)p:nth-of-type(2)父级下第二个 p 标签
    :nth-last-of-type(n)p:nth-last-of-type(3)父级下倒数第三个 p 标签

  2. 第二组

    结构选择器示例说明
    :first-childspan:first-child父级下第一个 span 子元素
    :last-childspan:last-child父级下最后一个 span 子元素
    :nth-child(n)span:nth-child(2)父级下第二个 span 子元素
    :nth-last-child(n)span:nth-last-child(2n-1)父级下倒数的奇数 span 子元素

说明：

  • 关于 n：n 从 1 开始，n 可以取值为 2n 、 3n+1 、 even 、 odd 等。
  • 关于类名：
    :nth-child(n) 会先选中第 n 个子元素，且该子元素拥有此类时才会应用样式；
    :nth-of-type(n) 会先选中每一类型标签下的第 n 个元素，且当被选中的元素拥有此类时才会应用样式。

# 三大特性

# 层叠性

相同选择器设置相同的样式，此时一个样式会覆盖另一个冲突的样式。
哪个样式离结构近，就执行哪个样式。

# 继承性

子标签会继承父标签的某些样式，如颜色、字号和行高等。恰当使用可降低代码复杂性。

# 优先级

选择器选择器权重 (a,b,c,d)
继承或者 *0,0,0,0
元素选择器0,0,0,1
类 / 伪类选择器0,0,1,0
id 选择器0,1,0,0
内联样式1,0,0,0
!important无穷大

权重比较：从左往右依次，较大的权重大，如果相同，则继续往右移动比较。
权重叠加：复合选择器使用时间会有权重叠加，需要计算权重，但是权重不会进位。

# 字体属性

属性说明备注
font-size字号取值数字 + px 或者关键字 (small/medium/large)
font-family字体多字体从前往后依次采用，“字体 1”,“字体 2”,…,“字体 n”
font-weight字体粗细取值 100~900 或关键字 (lighter/normal/bold)
font-style字体样式斜体效果，取值 italic/oblique
font字体连写连写顺序不能更换，必须写字号和字体

font 连写顺序：font: font-style font-weight font-size/line-height font-family

# 文本属性

属性说明备注
color文本颜色取值为关键字、十六位进制或 RGB 值
text-align文本对齐取值为 left/center/right (对 inline/inline-block 元素均有用)
text-indent文本缩进取值为像素值，两字距缩进：2em 或字号的 2 倍 px
text-decoration文本修饰取值 none (无划线)/underline (下划线)/line-through (中划线)/overline (顶划线)
text-transform文本大小写取值 none (无)/uppercase (大写)/lowercase (小写)/capitalize (首字母大写)
text-spacing字间距取值为像素值 (每个英文字母视为一个 "字")
word-spacing词间距取值为像素值 (每个英文单词)
line-height行间距取值为像素值
vertical-align垂直对齐取值为 top/middle/bottom/baseline (基线)

# 背景属性

属性说明值
background-color背景颜色取值为关键字、十六位进制或 RGB 值
background-image背景图片取值为 url (图片路径)
background-repeat是否平铺repeat/no-repeat/repeat-x/repeat-y
background-position背景位置取值为像素值，分别为 X 和 Y 坐标
background-attachment背景附着取值 scroll (滚动)/fixed (固定)
背景简写简化书写顺序：颜色、地址、平铺、滚动、位置
背景色半透明背景颜色半透明background:rgba (0,0,0,0.3); 后面必须是 4 个值，最后一个为透明度

# CSS 盒子模型

在 CSS 中，所有 HTML 元素可以看作盒子，一个用盒模型（box model）封装的 HTML 元素，包括边距，边框，填充，和实际内容四个部分。

# 盒子模型

在标准盒模型 (content-box) 中，为元素设置 width 和 height 实际是设置 content 的宽度和高度。

# 边框 (border)

属性说明值
border-width边框大小取值为像素值
border-style边框样式solid (实线)/dashed (虚线)/dotted (点线)
border-color边框颜色取值为颜色取值
border边框简写形式width style color
border-collapse边框合并separate (默认值，独立边框)/collapse

取消边框： border: 0 或者 border: none ，两者的区别在于有渲染或没渲染。

# 内边距 (padding)

  1. 内边距可以每个方向单独指定： padding-left 、 padding-right 、 padding-top 、 padding-bottle
  2. 内边距也可以简写，根据值的多少应用：
    padding: 1个值(四周)/2个值(上下&左右)/3个值(上&左右&下)/4个值(上右下左)
  3. 一般情况下，如果盒子本身没有指定 width/height，padding 不会撑大盒子。

# 外边距 (margin)

  1. 外边距的设置方式与内边距完全相同，包括单独设置和简写方式。
  2. margin 相关：
    相关情况说明
    margin 塌陷父子关系的块元素，同时有外边距时，父元素会取两者中较大的
    margin 合并兄弟关系的块元素，同时有外边距时，相邻处外边距会取两值中较大的
    负 margin设置负值的 margin 时，元素会被 “拉” 向负值方向

# 块级盒子与内联盒子

# 特性对比

块级盒子 (block)内联盒子 (inline)
块级盒子独占一行相邻内联盒子占一行，一行内可以有多个
高度、宽度、内外边距可以控制高度，宽度直接设置无效，默认宽度是本身内容宽度
内部可以放其他块级盒子和内联盒子内部只能容纳其他内联盒子
div、p、dt、h1 等标签默认都是 block 盒子
但 p、dt 和 h1 等标签内不能容纳其他块级盒子		a 标签、span 标签默认是 inline 盒子

# 行内块元素

行内块元素 (inline-block)，又称可变元素，兼顾 block 盒子和 inline 盒子特点：

  • 默认宽度为内容宽度，一行内可以放置多个；
  • 可以指定宽度、高度和内外边距；
  • img、input、textarea 默认显示为 inline-block。

# display 属性

通过 display 属性可以指定元素的显示方式。

display: none 隐藏元素后不占用原来的位置；
visibility: hidden 隐藏元素后占用原来的位置。

# CSS 浮动

# 正常文档流

正常布局流、标准流 (normal flow)：

  • 默认的布局方式。
  • 块级元素独占一行，按书写顺序从上往下放置。
  • 内联元素与其他内联元素和文本同处一行，从左往右依次排满，如果一行空间不够则换行。

# 浮动与特性

浮动 (float)：创建浮动框，将其移动到一边直至碰到块元素或另一个浮动框的边缘。
特性：

  • 浮动的元素会脱离标准流 (脱标)，移动到指定位置，并且不再保留原先的位置。
  • 浮动的元素会在一行内显示并且元素顶部对齐。
  • 浮动的元素会具有行内块元素的特性。
  • 浮动的元素只影响后面的标准流，不会影响前面的标准流。

# 布局说明

  • 多个块级元素，纵向排列标准流，横向排列用浮动。
  • 一个盒子里有多个盒子，如果其中一个盒子浮动了，其他兄弟也应该浮动。
  • 为了防止浮动带来的影响，使用浮动后应清除浮动。

# 清除浮动

浮动元素不再占用原文档流的位置，会对后面的元素排版产生影响，为了解决这些问题，就需要清除浮动。

# CSS 定位

CSS 元素定位由定位模式与边偏移两个部分组成。
定位模式：static、relative、absoulute、fixed、sticky
边偏移：top/bottom/left/right，指定位元素相对父元素边线的距离。

# 定位模式

  1. 静态定位 (static)
    元素默认定位方式，没有边偏移。
  2. 相对定位 (relative)
    • 元素在移动位置时，是以原来的位置为参考点。
    • 不脱标，仍然以标准流的方式占用原来位置。
  3. 绝对定位 (absolute)
    • 元素无祖先元素或祖先元素无定位，以浏览器为参考点。
    • 元素祖先元素有定位，则以最近一级有定位的祖先元素为参考点。
    • 绝对定位元素脱标。
  4. 固定定位 (fixed)
    • 元素固定于浏览器可视区位置，浏览器页面滚动时，元素位置不变。
    • 固定定位元素脱标，不占原先位置，跟父元素定位无关。
  5. 粘性定位 (sticky)
    • 元素以浏览器可视窗口为参照点移动。(固定定位特点)
    • 粘性定位元素占用原先的位置。(相对定位特点)
    • 必须添加 left/right/top/bottom 其中一个才有效果。

# 定位的特点

  • 脱标的定位元素有行内块元素的性质，脱标的定位元素不会触发外边距塌陷。
  • 定位的盒子有 z-index 属性，越大的层级可以压住小的层级。
  • 浮动的盒子不会压住下面盒子中的内容，定位的盒子会压住下面盒子的内容。

# BFC 及应用

# BFC 概念

块格式化上下文（Block Formatting Context，BFC）是 Web 页面的可视 CSS 渲染的一部分，是块级盒子的布局过程发生的区域，也是浮动元素与其他元素交互的区域。
简单来说，具有 BFC 特性的元素可以看作是隔离了的独立容器，容器里面的元素不会在布局上影响到外面的元素，并且 BFC 具有普通容器所没有的一些特性。

# BFC 特点

  1. BFC 区域相当于一个容器，内部元素不会影响到外部，外部元素也不会影响到内部。
  2. 同一个 BFC 区域内，外边距会发生折叠。
  3. BFC 区域内的元素不会与浮动元素重叠。
  4. 计算 BFC 区域的高度时，浮动元素也参与计算。

# 创建 BFC

  1. 根元素 ( html )(html 所在区域即为 BFC 区域，因此默认整个网页都在 BFC 内渲染)
  2. 浮动元素 ( float 不是 none )
  3. 绝对定位元素 ( position 为 absolute 或 fixed )
  4. 行内块元素 ( inline-block )、弹性元素 ( flex 、 inline-flex )、网格元素 ( grid 、 inline-grid ) 等…
  5. overflow 不为 visible 的块元素
  6. 其他

# BFC 应用

  1. 消除 margin 塌陷

父子元素的 margin 塌陷也可以通过给父元素设置 overflow 属性来解决。

  1. 清除浮动
父元素高度被浮动盒子撑开
  1. 防止浮动元素重叠

# 布局整体思路

先结构，后样式

  1. 测量确定确定页面版心 (可视区)。
  2. 分析页面行模块以及每个行模块中列模块。(布局第一准则)
  3. 分析确定每个列模块大小，再确定列模块位置。(布局第二准则)
  4. 制作 HTML 结构，确定各盒子在页面的位置。(普通流 + 浮动 + 定位)
  5. 确定结构后，设置盒子样式，往盒子中添加内容。
