# Document

# 浏览器环境

JavaScript 语言最初是为浏览器创建的，但在经过不断发展后，它已经不局限于浏览器环境中。
运行 JavaScript 的平台称为主机，特定平台的功能称为主机环境。
一个完整的 JavaScript 实现包含了三个部分：核心（ECMAScript）、文档对象模型（DOM）和浏览器对象模型（BOM）。
文档对象模型（Document Object Model），简称 DOM，将所有页面内容表示为可修改的对象。 document 对象是页面主要的 “入口点”。
浏览器对象模型（Browser Object Model），简称 BOM，表示由主机环境（如浏览器）提供的用于处理文档（document）之外所有内容的其他对象。
在 “广义的 HTML” 规范中，DOM 和 BOM 都是它的一部分。

# DOM 树

HTML 文档的主干是标签（tag）。
根据 DOM，每个 HTML 标签都是一个对象，嵌套的标签是闭合标签的子标签（children）

# 一个例子

这个 HTML 文档的 DOM 基本如下：

flowchart TD
    html --> head
    html --> body
    head --> meta
    head --> title
    body --> h1
    body --> p1
    body --> p2

每个树的节点都是一个对象。如 <html> 是根节点， <head> 和 <body> 是其子节点，它们之中又有其他子节点。
标签被称为元素节点，元素内的节点形成文本节点。文本节点被标记为 #text ，一个文本节点只包含一个字符串，它没有子项并总是叶子节点。

元素之间的换行和空格也会被列为文本节点，如 <head> 后， <body> 前的一个换行符。
通常情况下，因为它们无关紧要并通常不会影响文档显示方法，一些工具都会把它们隐藏。

# 一些规范

  • <head> 标签之前的空格和换行符均会被忽略，根据规范，如果在 <body> 之后放置了其他内容，它也会被移到 <body> 标签内的最下方
  • 遇到不正确的 HTML，在形成 DOM 时会自动更正它。如：
    • <html> 始终应该是根节点，如果 HTML 文档中缺少它，浏览器会自动创建它作为根节点
    • 没有关闭的标签也会关闭形成一个正常的节点
    • 表格 <table> 会自动创建 <tbody>
  • HTML 中所有内容，连同注释都会成为 DOM 的一部分，注释也是一种节点
  • 整个文档的 document ，甚至 <!DOCTYPE html> 指令也是一个 DOM 节点，规范中的节点一共有 12 种类型

# DOM 的结构

对 DOM 的所有操作都是以 document 对象开始的，它的 DOM 的 “入口”。
最顶层的树节点（即 html 节点）可以直接作为 document 的属性来使用。
<html> = document.documentElement ，这是最顶层 DOM 节点，也即 <html> 标签。
<head> 和 <body> 标签也是常访问的节点，分别可以使用 document.head 和 document.body 访问到。

# 子节点与子孙节点

子节点（childNodes）通常是对应直系子元素，子孙节点是嵌套元素中的所有元素，包括子节点和子节点的子节点。

  • childNodes 可以访问到包括文本节点在内的所有子节点
  • firstChild 和 lastChild 是访问第一个和最后一个子节点的方法
  • hasChild 方法可以判断是否具有子节点

childNodes 返回一个类数据的可迭代 DOM 集合，既可以使用 for..of 来迭代，也可以使用 Array.from 把它转变成一个真正的数组。
DOM 集合有两个性质，一是 DOM 集合是只读的，二是 DOM 集合是实时的。

# 兄弟节点和父节点

flowchart TD
html --> head
html --> body
head --> meta
head --> title
body --> h1
body --> p1
body --> p2

在这个 DOM 树中， <h1> 标签和 <p> 标签互为兄弟节点， <body> 标签是它们的父节点。

  • nextSibling 和 previousSibling 可以方便访问下一个兄弟节点和上一个兄弟节点
  • parentNode 访问父节点

# 访问元素

正如之前介绍，所有 HTML 内容都会称为 DOM 的一部分，节点中也包含了文本节点和注释节点，而多数情况下，需要访问的仅仅是元素节点。这些方法是访问元素节点方法：

  • children ：所有子元素节点
  • firstElementChild 和 lastElementChild ：第一个和最后一个子元素节点
  • nextElementSibling 和 previousElementSibling ：下一个和下一个兄弟元素节点
  • parentElement ：父元素节点

# 搜索方法

  • document.getElementById(id) ，对于具有 id 属性的元素获取该元素，因为 id 是全页面独一无二的
  • querySelectorAll(selector) ，返回与给定 CSS 选择器相匹配的所有元素
  • querySelector(selector) ，返回给定 CSS 选择器相匹配的第一个元素，按嵌套顺序从外层到内部
  • matches(selector) ，检查元素是否与给定 CSS 选择器相匹配，返回 true/false
  • closest(selector) ，查找并返回给定 CSS 选择器相匹配的第一个祖先，元素自己也会被搜索。
  • getElementsByName 、 getElementsByClassName 、 getElementsByTagName ，分别按照 name ， tag 和 class 查找符合的元素并返回一个可迭代的类数组对象

比较 querySelectorAll 与 getElementBy* 方法：

  • 两者返回的都是一个可迭代的类数组对象，不同的是 querySelectorAll 方法中返回的是静态的 —— 它不会随着文档元素的更新而变化，而 getElementBy* 方法返回的是动态的对象，会实时反应文档元素的变化。
  • querySelectorAll 方法更灵活，写法也更多，现在的使用情况更多。

# 节点属性

# DOM 节点类

每个 DOM 节点都属于相应的内建类。在层次结构中，根节点是 EventTarget ， Node 继承自它，其他节点又全继承于 Node 。

flowchart BT
HTMLInputElement --> HTMLElement
HTMLBodyElement --> HTMLElement
HTMLAnchorElement --> HTMLElement
HTMLElement --> Element
HTMLDocument --> Document
Text --> CharacterData
Comment --> CharacterData
CharacterData --> Node
Element --> Node
Document --> Node
Node --> EventTarget
  • EventTarget ：一切的根抽象（abstract）类，该类的对象从未被创建，它是为了让所有 DOM 节点都能支持事件（event）
  • Node ：一个抽象类， Node 类的对象从未创建，但是它提供了 DOM 树的核心功能，是所有 DOM 节点的基础
  • CharacterData ：一个抽象类，被 Text （元素内部文本的类）和 Comment （注释类）继承
  • Document ：被 HTMLDocument 继承，作为全局 document 的 DOM 入口
  • Element ：DOM 元素的基础类，提供元素级导航方法和搜索方法， Element 为 XML 和 SVG 等更具体类提供基础
  • HTMLElement ：是所有 HTML 元素的基础类，被其他更具体的 HTML 元素继承

一个普通的 DOM 元素节点，它的全部属性和方法都是继承链的结果，并且追根溯源，继承自 Object ，因此对象的方法也是有效的：

# 常见节点属性

  1. nodeType
    nodeType 提供了一种获得 DOM 节点类型的方法，它返回一个数值型值。

    • 对于元素节点， elem.nodeType == 1
    • 对于文本节点， elem.nodeType == 3
    • 对于 document 对象， elem.nodeType == 9 ，… 还有其他节点对应的数值

  2. nodeName 和 tagName
    tagName 仅针对元素节点，返回标签名， nodeName 适用于所有 Node ，在读取元素节点时，它们相同。

  3. innerHTML
    innerHTML 属性将元素中的 HTML 获取为字符串形式，通过它可以更改页面。
    innerHTML += 会进行完全的重写：先移除全部旧的内容，再用添加了新内容的 HTML 重写页面，页面全部的图片和其他资源都需要重新加载。

  4. outerHTML
    outerHTML 属性包含了元素完整的 HTML，等同于 innerHTML 加上元素本身。

    和 innerHTML 不同，写入 outerHTML 不会改变元素，而是在 DOM 中替换掉它。
    div.outerHTML = ... 会将旧的标签从文档中移除，然后将另一个新的 HTML 片段插入到它的位置上，原来的标签仍然具有其旧的值。

  5. nodeValue/data
    文本节点内容，读取文本节点的信息， innerHTML 属性是只适应于元素节点的。

  6. textContent
    访问元素内部的纯文本内容，所有标签都会被去掉。
    写入 textContent 比写入 innerHTML 安全得多，所有符号都会按字符串处理而不会被当作 HTML 片段。

  7. hidden
    当设置为 true 时，与 CSS 的 display: none 执行相同的事。

# 特性与属性

页面加载时，会解析 HTML 并从中生成 DOM 对象，大多数标准 HTML 的特性（attributes）会自动变成 DOM 对象的属性（properties）。
例如 <body id = "page"></body> 的 DOM 对象就会有 body.id = "page" ，但是特性与属性并不是一一对应的，有些情况下它们也会有所区别。

  1. DOM 属性
    因为 DOM 是对象，所以按照一般对象的操作对它来说都是可行的，包括创建新属性，添加方法等。

    • 属性的值可以有不同的数据类型
    • 属性是大小写敏感的

  2. HTML 特性
    HTML 标签可能会拥有特性，当进行解析时，标准的特性会被创建为 DOM 对象的属性一一对应，非标准的特性不会。

    • 特性的值总是字符串类型
    • 特性是大小写不敏感的
    • 设置的特性在 outerHTML 都是可见的

    当一个标准的特性被改变，对应的属性也会自动更新，（除了几个特例）反之亦然。
    一些值只允许特性到属性的改变，改变属性值不会改变特性值，因为这些属性值可能与用户行为有关，是为了防止用户改变特性值。

  3. 非标准特性
    所有以 date-* 开头的特性都被保留给程序员使用，以避免冲突，它们可以在 dataset 中使用。
    使用 data-* 特性是一种合法且安全的传递自定义数据的方式。

# 修改文档

# 修改文档的方法

  1. 创建节点

    • document.createElement(tag) ：用给定标签创建一个元素节点
    • document.createTextNode(value) ：创建一个文本节点
    • elem.cloneNode(deep) ：克隆元素， deep = true 则连同后代一起克隆

  2. 插入方法

    • node.append(nodes/strings) ：在节点末尾插入
    • node.prepend(nodes/strings) ：在节点开头插入
    • node.before(nodes/strings) ：在节点之前插入
    • node.after(nodes/strings) ：在节点之后插入
    • node.replaceWith(nodes/strings) ：替换节点
    • node.remove() ：移除节点

    这种方法插入的只能是节点或者是字符串，即使是 node.append(<p></p>) 这种形式，其中的文本也全部都会作为字符串插入。
    如果插入已经存在页面的一个节点，节点会自动从原本的位置被删除，而插入到新的位置。

    elem.insertAdjacentHTML(where, html) 方式可以插入一段 HTML 字符串。
    第一个参数确定插入的位置，第二个参数中的字符串全都会作为 HTML 插入。

  3. DocumentFragment
    DocumentFragment 方法是一个特殊的 DOM 节点，用来传递节点列表的包装器（wrapper）。

  4. insert/remove

    • parentElem.appendChild(node) ：将 node 附件为 parentElem 最后一个子元素
    • parentElem.insertBefore(node, nextSibling) ：在 parentElem 的 nextSibling 前插入 node
    • parentElem.removeChild(node) ：从 parentElem 中删除子元素 node
    • parentElem.replaceChild(newElem, node) ：将 parentElem 后代中的 node 替换成 newElem
    • document.write(html) ：马上向页面写入 HTML 片段

    document.write(html) 不涉及 DOM 修改，且只在页面加载时工作。
    其他的操作 DOM 的方法在实际开发中已经可以用更新的方式替换。

# 样式和类

JavaScript 既可以修改 CSS 类，也可以修改 style 属性，但是 CSS 类应该是修改样式时的第一选择，仅当类无法处理时才应该考虑修改 style 样式。

# className 和 classList

className 对应 class 属性，可以读取更改 class ;
classList 是一个特殊的对象，它具有单个的 add/remove/toggle 方法，当需要添加、修改、删除类的时候，它具有更多的灵活性。

  • add|remove(class) ，添加 | 删除一个 class
  • toggle(class) ，如果一个类不存在则添加它，如果存在就删除它
  • contains(class) ，检查是否存在类，返回 true|false

# 元素样式

elem.style 属性是一个对象，对应 style 中的内容。
elem.style.width 或 elem.style.backgroundColor ，多词使用驼峰式写法。

# 读取样式

getComputedStyle(element, [pseudo]) 方法可以获取具有样式属性的对象，这里获得的是最终元素的解析属性。

计算（computed）样式是所有 CSS 规则和 CSS 继承都运用后的结果。
解析（resolved）样式是最终应用于元素的样式值。
getComputedStyle 中返回的是解析值。

# 元素大小和滚动

  • offsetParent 和 offsetLeft/Top
    offsetParent 是最接近的祖先元素，浏览器渲染期间用它来计算坐标，优先级按 CSS 中定位标准。
    offsetLeft/Top 提供元素相对祖先元素左上角的 X/Y 方向的位移坐标。
    对 body/html 标签， offsetParent 的值为 null 。

  • offsetWidth/Height
    元素外部的宽度和高度，元素的完整大小（包含边框）。
    如果一个元素未显示，则其为 0。其 offsetParent 值为 null 。

  • clientTop/Left 和 clientWidth/Height
    clientTop/Left 是元素上边框和左边框的宽度，当有边框时会包括边框的宽度。
    clientWidth/Height 是元素边框内区域大小，等同于 content + padding 区域。

  • scrollWidth/Height 和 scrollLeft/scrollTop
    scrollWidth/Height 是内容区域完整的大小，包含滚动出去的部分。
    scrollLeft/scrollTop 是元素隐藏，滚动部分的，即已经滚动了多少。

# 窗口大小

  • document.documentElement.clientWidth ：获取当前窗口的宽度
  • window.innerWidth/innerHeight ：属性值的宽 / 高包括了滚动条占用的部分

# 文档滚动

  • window.pageXOffset ：水平方向（从左侧开始）的滚动距离
  • window.pageYOffset ：垂直方向（从顶部开始）的滚动距离
    这两个名称分别也是 window.scrollX 和 window.scrollY 的别名。

必须要在 DOM 完全构建好之后才能通过 JavaScript 滚动页面，否则它不会正常工作。

  • window.scrollBy(x, y) ：将页面滚动到相对于当前位置 x 和 y 的距离，单位是 px

  • window.scrollTo(x, y) ：将页面滚动到绝对坐标，相当于分别设置 (pageX, pageY)

  • elem.scrollIntoView(top) ：将页面滚动使其顶端与 elem 顶端对齐， top 取值为 false 则会将元素顶端于页面底部对齐

    设置 document.body.style.overflow = "hidden" 禁止页面滚动，去除这个属性恢复页面滚动。

# 坐标

元素坐标有两种：

  • clientX/Y ：相对于窗口，类似 position:fixed
  • pageX/Y ：相对于文档，类似 position:absolute

# 窗口坐标

elem.getBoundingClientRect() 返回最小矩形的窗口坐标， elem 作为内建 DOMRect 类的对象。

与 CSS 中不同，JavaScript 中 right 和 bottom 不是表示从右或者底开始计算，元素所有坐标都是从左上角开始计算的。
document.elementFromPoint(x, y) 会返回在窗口坐标内嵌套最多的元素，因为是窗口坐标，所以元素可能会随着窗口滚动变化，对于窗口之外的坐标会返回 null 。
文档坐标可以结合固定定位使用。

# 文档坐标

JavaScript 没有直接获取文档坐标的方法，但文档坐标可以与窗口坐标建立关联：

  • pageX = clientX + 文档水平滚动出的部分的宽度
  • pageY = clientY + 文档垂直滚动出的部分的高度

文档坐标可以结合绝对定位使用。
