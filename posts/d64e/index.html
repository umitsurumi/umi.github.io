# 链表的数据结构

# 数组

数组是最常用来存储多个元素的数据结构，数组可以实现随机存取，通过下标 index 在O(1)O(1)O(1) 的时间复杂度中访问元素。
但是在大多数语言中，数组的大小是固定的，而在数组的起点和中间插入或移除元素的时间成本也较高。

# 链表

链表存储有序的元素集合，但不同于数组，链表中的元素在内存中并不是连续放置的。每个元素由一个存储元素本身的结点和一个指向下一个元素的引用（也称指针或链接）组成。
相对于传统的数组，链表的好处在于，它的长度并不是固定的，方便后续增加新的元素，同时在添加或移除元素的时候不需要移动其他元素。

flowchart LR
head --> D1((data))-->D2((data))-->D3((data))-->null

# 双向链表

单向链表只能从头依次顺序地向后遍历，双向链表的每个结点存有 prior 和 next 两个指针，一个指向它的前驱结点，一个指向它的后继结点。因此双向链表既可以很容易找到当前结点的前驱结点。

# 双链表结点

双链表和单链表最大的区别在于它的每个结点还有指向上一个结点的指针。
就像表尾结点的 next 指针指向 null 一样，双链表表头结点的 prior 指针也指向 null 。

flowchart LR
head --> id1((DuNode)) --next-->id2((DuNode))--next--> id3((DuNode))--next--> idEnd(null)
id3((DuNode))--prior-->id2((DuNode)) --prior--> id1((DuNode)) --prior-->  idStart(null)

# 循环链表

循环链表可以是单链表也可以是双链表，与这两种链表的区别在于，如果是循环单链表，那么它的最后一个结点的 next 指向的是第一个结点；而如果是循环双链表，那么它的第一结点的 prior 指针指向它的最后一个结点，而它的最后一个结点的 next 指针也是指向它的第一个结点

循环单链表

flowchart LR
head-->id1((data1))--next-->id2((data2))--next-->id3((data3))--next-->id1

循环双链表：

flowchart LR
head-->id1((data1))--next-->id2((data2))--next-->id3((data3))--next-->id1
id3--prior-->id2--prior-->id1--prior-->id3
