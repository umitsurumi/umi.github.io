# 集合

集合（Set），简称集，和数字概念中的集合类似，该数据结构由一组无序且唯一的元素组成。
在 ECMAScript2015 中，Set 类已经成为 JavaScript 的标准内置对象，而在这之前，可以通过 Object 来模拟实现。

作为标准内置对象的 Set 类同样也有类似上述的这些方法，但是稍有区别：

  1. Set() 构造函数可以接受一个数值，或者一个可迭代对象，来创建集合。
  2. .values() 和 .keys() 两者完全相同，返回的是一个迭代器对象，它按插入顺序包含集合中每个元素的值。
  3. .forEach() 方法可以对集合中的每个值按插入顺序执行一次提供的函数。

# 集合运算

集合可以进行以下的运算：

运算说明
并集对于给定的两个集合，返回一个包含两个集合元素的新集合
交集对于给定的两个集合，返回一个包含两个集合中共有元素的新集合
差集对于给定两个集合，返回一个包含所有存在一集合而不存在二集合的元素的新集合
子集验证一个集合是否为另一个集合的子集

# 映射

集合关注的主要是值本身，而在映射（Map）中，则采用键值对的形式来存储数据。
JavaScript 中的 Object 类是采用键值对形式存储数据的类型，但是能作为键的数据类型只能是 string 或者 symbol 。ECMAScript2015 将 Map 类加入到 JavaScript 的标准内置对象中，在写入删除密集的情况下，它有更好的性能表现。
Map 与 Object 区别见下：

区别说明
意外的键Map 只有显示插入的键，而 Object 有一个原型
键的类型Map 的键可以是任意值，而 Object 的键只能是 string 或 symbol 类型
键的顺序Map 的键按插入顺序，而 Object 的键按数字到字符串顺序
长度Map 有 size 属性获取长度，Object 需要手动统计
性能Map 在频繁增删键值对的场景下表现更好

# 方法与属性

方法或属性说明
sizeMap 的属性，等于键值对的数量
clear()删除 Map 中所有元素
delete(key)删除 Map 中指定元素
get(key)根据键返回指定的元素
has(key)判断指定键的元素是否存在
set(key, value)指定键值对
forEach()按照插入顺序对 Map 中每个键值对执行一次给定的函数

另外还有 keys() 、 values() 和 entries() 方法，它们返回的都是一个迭代器对象，分别包含每个元素的 key 、 value 或 [key, value] 对。

# 散列表

散列表，也称为哈希表（HashTable、HashMap），是通过键值（key）直接访问对应数值（value）的一种数据结构。它通过计算出一个键值的函数，将所需查询的数据映射到表中一个位置来让人访问，这加快了查找速度。这个映射函数称做散列函数，存放记录的数组称做散列表。

# 冲突处理

有些时候，一些键会有相同的散列值，这与散列函数的设计有关。不同的值在散列表中对应相同位置的时候，称其为冲突。
例如，如果散列函数设计为 hash = key % 7 ，当 key 值为 8、15 和 22 时，它们的散列值都为 1。
常用解决冲突的方法有两种，分离链接法或线性探测法。

# 分离链接法

分离链接法包括为散列表的每一个位置创建一个链表并将元素存储在里面，它是解决冲突的最简单的方法。如下图所示：

相应的各方法也需要进行改写：

  • put 方法，如果是第一次在该散列值对应的位置放入元素时，应该在该位置初始化一个单链表，当遇到冲突后，冲突的元素继续在链表上进行添加。
  • get 方法，不仅需要根据散列值找到对应的元素所在的链表，同时还需要检查链表每个结点中的 key 值是否等于查找的 key 值，然后来返回相应的结果。
  • remove 方法，如果元素删除后，链表非空，可以直接使用链表的删除方法。如果删除元素后链表为空，还需要将对应的散列值删除（或者直接置空）。

# 线性探测法

另一种解决冲突的方法是线性探查。之所以称作线性，是因为它处理冲突的方法是将元素直接存储到表中，而不是在单独的数据结构中。
当想向表中某个位置添加一个新元素的时候，如果索引为 position 的位置已经被占据了，就尝试 position+1 的位置。如果 position+1 的位置也被占据了，就尝试 position+2 的位置，以此类推，直到在散列表中找到一个空闲的位置。如下图所示：
